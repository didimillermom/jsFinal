Dom Manipulation Using createElement and appendChild

Dom stands for Document Object Module. This is a representation of the web page or HTLM that you are working with. It is kind of like a family tree with each of the objects of an HTML page represented. Attached is an image from wikipedia showing and example of DOM Tree: The top Element is the actual HTML Document and is considered to be the root node, next down are the Elements nodes, Text nodes and Attribute nodes. According to MDN a node is simply the interface of the element in the DOM.


So, when we are talking about modifying the DOM we are talking about modifying objects in an HTLM document. The DOM is not actually part of JavaScript but is a built-in API(application programming interface).

W3 School explains many of the DOM manipulation types and defines createElement "Creates an Element node". It also states that appendChild "Adds a new child node, to an element, as the last child node". As you can see in the previous image, element nodes are part of the tree that are in blue. The child elements would be text nodes or in our case the Element is the UL and the child is the li. Elements are things like "body, paragraphs, lists,and fonts.

The first thing we will try to do is create a new list item in the Unordered list that has been created in HTML. We will use both the createElement and appendChild methods to change the child of the UL.
